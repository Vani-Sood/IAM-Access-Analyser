# IAM Access Analyser

A multi-cloud IAM policy analysis platform powered by Google Gemini. Paste a policy, run a live scan, or hit the REST API — get findings, risk graphs, and fix suggestions in seconds.

---

## Features

- **Multi-cloud analysis** — AWS, Azure, GCP, OCI, Alibaba Cloud (JSON paste or live scan)
- **AI-powered findings** — Gemini identifies overly permissive rules, privilege escalation paths, and remediation steps
- **Interactive graph** — D3 visualisation of permission relationships
- **REST API** — API key auth, rate limiting, webhook delivery on analysis events
- **Audit log** — Tamper-evident event log with SHA-256 hashes, admin-only
- **Celery + Redis** — Async analysis workers with job polling
- **Neo4j** — Graph database for deep permission path queries

---

## Stack

| Layer | Tech |
|-------|------|
| Backend | FastAPI + SQLAlchemy + Alembic |
| Workers | Celery + Redis |
| Databases | PostgreSQL (relational) · Neo4j (graph) |
| AI | Google Gemini (`gemini-2.5-flash-lite` default) |
| Frontend | Vanilla JS + D3.js (served as static files) |
| Auth | JWT (session) · API Keys (`ApiKey` prefix) |
| Cloud SDKs | boto3 (AWS) · azure-identity (Azure) · google-api-python-client (GCP) |

## Quick Start (Docker)

### 1. Clone and configure

```bash
git clone https://github.com/Vani-Sood/IAM-Access-Analyser.git
cd IAM-Access-Analyser
make setup          # copies .env.example → .env and generates JWT_SECRET
```

Edit `.env` and fill in required values:

```env
# Required
GEMINI_API_KEY=your-gemini-api-key
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=changeme

# Auto-generated by make setup
JWT_SECRET=

# Optional — enable live cloud scanning
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=us-east-1

AZURE_TENANT_ID=
AZURE_CLIENT_ID=
AZURE_CLIENT_SECRET=
AZURE_SUBSCRIPTION_ID=

GCP_PROJECT_ID=
GCP_SERVICE_ACCOUNT_KEY_JSON=
GOOGLE_APPLICATION_CREDENTIALS=
```

### 2. Start

```bash
make up
```

Opens at **http://localhost:8000**

### 3. Log in

Use the `ADMIN_EMAIL` / `ADMIN_PASSWORD` you set in `.env`.

## API Usage

### Authenticate

**Session JWT** (from browser login):
```
Authorization: Bearer <jwt>
```

**API Key** (for CI/CD, scripts):
```
Authorization: ApiKey <your-key>
```

Generate API keys in **Settings → API Keys**.

### Analyse a policy

```bash
curl -X POST http://localhost:8000/api/v1/analyze \
  -H "Authorization: ApiKey <your-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "mode": "json",
    "cloud": "aws",
    "policy": {
      "Version": "2012-10-17",
      "Statement": [{
        "Effect": "Allow",
        "Action": "*",
        "Resource": "*"
      }]
    }
  }'
```

Response includes an `analysis_id`. Poll for results:

```bash
curl http://localhost:8000/api/v1/analyses/<analysis_id> \
  -H "Authorization: ApiKey <your-key>"
```

### Live scan (AWS example)

```bash
curl -X POST http://localhost:8000/api/v1/analyze \
  -H "Authorization: ApiKey <your-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "mode": "live",
    "cloud": "aws",
    "role_arn": "arn:aws:iam::123456789012:role/MyRole"
  }'
```

## Webhooks

Register a webhook URL to receive POST notifications when analyses complete.

```bash
curl -X POST http://localhost:8000/api/v1/webhooks \
  -H "Authorization: ApiKey <your-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://hooks.slack.com/services/...",
    "event_type": "analyze.create"
  }'
```

Webhook payload:
```json
{
  "event": "analyze.create",
  "analysis_id": "...",
  "status": "completed",
  "timestamp": "2026-05-20T09:00:00Z"
}
```

## Environment Variables Reference

| Variable | Required | Description |
|----------|----------|-------------|
| `GEMINI_API_KEY` | Yes | Google Gemini API key |
| `ADMIN_EMAIL` | Yes | Bootstrap admin account email |
| `ADMIN_PASSWORD` | Yes | Bootstrap admin account password |
| `JWT_SECRET` | Yes | Secret for signing JWTs (auto-generated by `make setup`) |

| `REDIS_URL` | Auto | Set by docker-compose |

| `RATE_LIMIT_PER_HOUR` | No | Default: `1000` |
| `AWS_ACCESS_KEY_ID` | No | For live AWS scanning |
| `AZURE_TENANT_ID` | No | For live Azure scanning |
| `GCP_PROJECT_ID` | No | For live GCP scanning |
