API Keys

Keys authenticate API requests from CI/CD pipelines and external tools. Scoped to the current organization.

How to use an API key

Pass the key in the Authorization header with the ApiKey prefix:

Authorization: ApiKey <your-key>

curl example:

curl -X POST https://your-host/api/v1/analyze \
  -H "Authorization: ApiKey <your-key>" \
  -H "Content-Type: application/json" \
  -d '{"mode":"json","policy":{...}}'

Scopes:

  • read — GET endpoints only
  • readwrite — GET + POST (run analyses, manage resources)
  • admin — full access including org management
